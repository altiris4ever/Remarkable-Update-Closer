# reMarkable Update Closer

A small PowerShell utility that suppresses the update dialog generated by the **reMarkable Companion App**.

The reMarkable updater is built on the **WinSparkle** framework:
https://winsparkle.org

This script detects the update dialog window and closes it automatically.

⚠️ Note: This repository contains the **legacy method** for handling the updater.
A newer and more robust approach replaces the WinSparkle DLL entirely.

---

# Overview

The reMarkable Companion App uses the **WinSparkle updater**.
While the framework supports features like postponing or skipping updates, the implementation in the reMarkable client may repeatedly display update dialogs.

This script monitors open windows and automatically closes the update dialog when it appears.

---

# Features

## Core Functionality

* Identifies update dialog windows by **window title** and **process name**
* Enumerates open windows using the Windows **User32 API**
* Sends a `WM_CLOSE` message to dismiss the dialog

---

# Script Variants

## Remarkable-Update-Closer.ps1

Basic version.

Functionality:

* Enumerates all open windows
* Matches window title and process name
* Sends `WM_CLOSE` to terminate the update dialog

---

## Remarkable-Update-Closer.concept.ps1

Extended concept version with additional functionality.

Enhancements include:

* CPU optimization (dynamic polling interval)
* Process-aware monitoring
* Logging support
* Improved error handling using try/catch blocks

# Installation

## Requirements

* Windows PowerShell
* Optional: **PS2EXE** for compiling the script to an executable

---

# Compiling to EXE (optional)

You can compile the script into an executable for easier deployment.

## Install PS2EXE

Run PowerShell as Administrator:

```powershell
Install-Module -Name PS2EXE -Scope CurrentUser
```

## Compile the Script

```powershell
Invoke-PS2EXE -InputFile "Remarkable-Update-Closer.ps1" -OutputFile "Remarkable-Update-Closer.exe"
```

Optional graphical tool:

```
Win-PS2EXE.exe
```

Compile with **"Compile a graphical Windows program (-NoConsole)"** for silent operation.

## Example with icon and no console

```powershell
Invoke-PS2EXE .\Remarkable-Update-Closer.ps1 -NoConsole -IconFile "C:\Temp\Icon.ico"
```

# Usage

Example deployment scenarios:

### Scheduled task

Start the script automatically when the system starts or when the application is installed.

### Wrapper launcher

Create a wrapper script that launches:

* `reMarkable.exe`
* `Remarkable-Update-Closer.exe`

Then replace the original shortcut with the wrapper executable.

### Application virtualization

When using **App-V**, the script can be started via a user startup script that launches both:

* the application
* the update suppression script

This ensures the update dialog is closed even if the application is started directly from the installation directory.

---

# Limitations

This script only closes the update dialog window.

It does **not disable the updater itself**.
The dialog may reappear if the application triggers another update check.
